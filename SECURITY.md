# Security Policy

## Reporting a Vulnerability

If you discover a security vulnerability in gerbil-data, please report it by opening an issue on the [GitHub repository](https://github.com/shardzhang/gerbil-data/issues). Do not open a public issue for critical vulnerabilities.

Please include the following details:

- A description of the vulnerability
- Steps to reproduce the issue
- Potential impact
- Any suggested fix (if available)

We will acknowledge receipt within 48 hours and provide an estimated timeline for a fix.

## Scope

This security policy applies to all components of the gerbil-data project, including source code, build scripts, and documentation.
